## 核心逻辑（test.c）

### 任务结构

```c
LOS_TaskCreate("Task_DS0", Task_DS0, ...);   // 检测 KEY0 → 翻转 DS0
LOS_TaskCreate("Task_DS1", Task_DS1, ...);   // 检测 KEY1 → 翻转 DS1
```

两个任务优先级均为 5，栈大小 1024 字节。

### 按键处理（以 Task_DS0 为例）

```c
void Task_DS0(void)
{
    while (1) {
        if (KEY0 == 0) {              // 检测低电平
            LOS_TaskDelay(20);        // 消抖
            if (KEY0 == 0) {
                LED0 = !LED0;         // 翻转 LED
                printf("KEY0 pressed, DS0 is %s\r\n", LED0 ? "OFF" : "ON");
                while (KEY0 == 0)     // 等待释放
                    LOS_TaskDelay(20);
                LOS_TaskDelay(20);    // 释放消抖
            }
        }
        LOS_TaskDelay(20);            // 轮询间隔
    }
}
```

### 注意事项

1. **禁止使用 `delay_ms()`**：LiteOS 接管了 SysTick 定时器，裸机延时函数会与操作系统冲突。全部改用 `LOS_TaskDelay()`。
2. **未使用 `KEY_Scan()`**：该函数内部有 `static` 变量，多任务下状态混乱，改为直接在任务中用 GPIO 位带宏 (`KEY0`/`KEY1`) 读取。
3. **LED 低电平点亮**：`LED0 = 0` 亮，`LED0 = 1` 灭，串口打印已对应翻转显示。
4. **串口波特率 115200**，使用 `printf()` 即可输出。
