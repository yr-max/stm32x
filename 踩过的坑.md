# 实验一踩过的坑（Top 4）

## 1. 芯片型号配置错误

原始工程是 STM32F103ZE（大容量 HD），C8T6 是**中容量 MD**，宏和启动文件都不对。直接编译报 `stm32f10x.h(298): error #67`。

**修复**：编译宏 `STM32F10X_HD` → `STM32F10X_MD`，启动文件改 `md.s`，Device 改 STM32F103C8。

## 2. Keil Device 自动定义宏导致冲突

改了编译宏后仍报错——根因是 Keil Device 仍是 ZE，设备包自动定义 `STM32F10X_HD`，与手动定义的 `STM32F10X_MD` 同时生效，枚举语法错误。

**修复**：在 `stm32f10x.h` 加保护——当两者同时定义时 `#undef STM32F10X_HD`。

## 3. C8T6 无 PE 口，PB2 未引出

原始代码用 `PE3/PE4/PE5` 做按键和 LED，C8T6 没有 PE 口。PB2（BOOT1）在最小系统板上也未引出。

**修复**：LED1 `PE5→PB0`，KEY0 `PE4→PB1`，KEY1 从 `PE3→PB2→PA1`。

## 4. LiteOS 编程约束

- `delay_ms()` 不能用（SysTick 被 LiteOS 接管），全改 `LOS_TaskDelay()`
- `KEY_Scan()` 内部 `static` 变量在多任务中混乱，改直接读位带宏

# 实验二踩过的坑（Top 4）

## 1. 移植江协驱动需要 stm32f10x_conf.h

江协库的 OLED/MySPI 驱动用 `GPIO_Init`、`SPI_Init` 等标准库函数，但正点原子工程默认不启用标准外设库（`USE_STDPERIPH_DRIVER` 被注释）。直接移植编译会报 40+ 个 `identifier undefined`。

**修复**：
- 创建 `Libraries/CMSIS/stm32f10x_conf.h`（包含 `stm32f10x_gpio.h` / `stm32f10x_rcc.h` / `stm32f10x_spi.h` / `misc.h`）
- 在 `SYSTEM/sys/stm32f10x.h` 中取消注释 `#define USE_STDPERIPH_DRIVER`
- 在 Keil 工程中添加 `stm32f10x_gpio.c` / `stm32f10x_rcc.c` / `stm32f10x_spi.c` / `misc.c`

## 2. W25Q64 WaitBusy 中 LOS_TaskDelay 导致系统卡死

`W25Q64_WaitBusy()` 在等 Flash 擦写完成（~400ms）时若调用 `LOS_TaskDelay(1)` 让出 CPU，高优先级任务（Key 优先级 4 > Storage 优先级 6）会抢占。长按按键时 Key 任务不断触发 `g_storage_changed=1`，Storage 不断重新进入 WaitBusy，形成死循环，系统完全卡死。

**修复**：WaitBusy 使用**纯空循环**（`while (busy) timeout--;`），并在 `Task_Storage` 调用 `LimitStorage_Write` 前后用 `LOS_TaskLock()/Unlock()` 锁死调度，保证写入原子完成。

## 3. OLED 雪花与软件 I2C 被中断

WaitBusy 中的 `LOS_TaskDelay` 让出 CPU 时，Task_Display（优先级 5）可能执行 OLED 软件 I2C 刷新。虽然 OLED（PORTB）和 W25Q64（PORTA）在不同 GPIO 端口，但频繁的任务切换本身会打乱软件 I2C 的时序，导致 OLED 收到垃圾指令产生**雪花屏**。

**修复**：问题 2 的修复（WaitBusy 纯空循环 + TaskLock）同时解决了雪花。另外 OLED 上电延时需 ≥100ms（`for (i=0;i<800000;i++);`），延时放在 `OLED_I2C_Init()` 之前而非之后效果更好。

## 4. 硬件接线顺序易搞错

OLED 和 W25Q64 模块的物理排针顺序与 STM32 GPIO 不是一一对应：
- OLED（4 脚 GND/VCC/SCL/SDA）：GND 和 VCC 接面包板电源轨，只有 SCL/SDA 接 PB6/PB7
- W25Q64（6 脚 VCC/CS/DO/GND/CLK/DI）：VCC 和 GND 也是电源，CS/DO/CLK/DI 分别接 PA4/PA3/PA2/PA1

常见失误：把电源脚当 GPIO 接了（严重，OLED 不亮 / W25Q64 不工作）、DO 和 DI（MISO 和 MOSI）接反（W25Q64 读不出 ID）。

**建议**：接线时对着 `实验报告/refs/实验二_硬件映射.md` 逐个确认，先接电源再接信号。

# 实验四踩过的坑（Top 4）

## 1. 电机驱动模块型号误判：L298N vs TB6612FNG

看到红色小板、大芯片、16 个引脚，第一反应是 L298N，但实际是 **TB6612FNG**。两者引脚命名和功能完全不同：L298N 是 IN1/IN2/ENA/OUT1/OUT2/+12V；TB6612FNG 是 AIN1/AIN2/PWMA/AO1/AO2/STBY/VM/VCC。

**修复**：按 TB6612FNG 重新接线，并用 A 通道（AIN1→PB8, AIN2→PB9, PWMA→PB10），B 通道悬空。

## 2. TIM2_CH3 remap 值错误导致 PB10 无 PWM

想将 TIM2_CH3 从默认 PA2 映射到 PB10，错误使用了 `GPIO_PartialRemap1_TIM2`。这个 remap 只移动 CH1/CH2，CH3 仍留在 PA2，导致 PB10 完全没有 PWM 信号，电机不转且 OLED 上 M=100 也无输出。

**修复**：使用 `GPIO_PartialRemap2_TIM2` 才能将 TIM2_CH3 映射到 PB10。

## 3. TB6612FNG 使能脚和动力电源没接对

TB6612FNG 的 **STBY 必须接 3.3V**，否则整片驱动芯片处于待机状态，AIN/PWM 信号全白费；**VM 必须接 5V** 给电机供电，VCC 只给芯片逻辑供电，不能混淆。

**修复**：STBY→3.3V，VM→5V，GND 与 C8T6 共地。

## 4. 电机铜片接线与共地

直流电机是两个裸露铜片，没有杜邦线母头，必须用鳄鱼夹转接。常见问题：两个鳄鱼夹夹到同一个铜片、铜片氧化接触不良、或者 TB6612FNG GND 没和 C8T6 共地。

**修复**：AO1/AO2 分别夹住不同铜片；用螺丝刀刮一下铜片表面；从面包板 GND 拉一根线到 TB6612FNG 的 GND 保证共地。

# 实验三踩过的坑（Top 4）

## 1. ADC 通道与 W25Q64 SPI 引脚冲突

参考例程 `7-2 AD多通道` 用 PA0~PA3 做 4 路 ADC，但当前工程 PA1~PA4 已被 W25Q64 软件 SPI 全部占用（MOSI/SCK/MISO/SS）。不改通道直接移植会导致 ADC 和 Flash 互相干扰。

**修复**：温度用 PA0（ADC_Channel_0），光照用 PA5（ADC_Channel_5），完美避开所有已占用引脚。

## 2. 传感器信号方向与直觉相反

MH-Sensor-Series 模块采用分压下臂结构（`VCC→10K→AO→传感器→GND`）：温度升高 → NTC 阻值降低 → AO 电压降低 → ADC 值降低；遮光 → 光敏阻值升高 → AO 电压升高 → ADC 值升高。手捏热敏显示降温、遮光显示更亮——初看以为是 bug，其实是电路决定的。

**修复**：翻转 ADC 值（`4095 - raw`），温度用 NTC 公式转换，光照映射到 0~100%。

## 3. OLED 画面随机偏移（GPIOB 软件 I2C 干扰）

LED（PB0/PB5）和 KEY（PB1/PB11）操作 GPIOB 时，偶尔产生类似 I2C 的时序信号，SSD1306 误解析为命令，导致画面整体下移甚至"斩断"——上半截在屏底、下半截在屏顶。仅重置偏移寄存器（0xD3）不够，显示起始行寄存器（0x40~0x7F）也可能被干扰。

**修复**：新增 `OLED_ResetOffset()`，每次刷新前同时重置起始行（0x40）和偏移（0xD3+0x00），每 200ms 自动校准。

## 4. W25Q64 旧数据范围不兼容

实验二的光照上限范围是 0~1500（模拟数据），W25Q64 里存了旧值（如 500）。实验三光照改为 0~100%，上电读出后 OLED 显示 `LL:437%`——旧数据在新范围完全非法。

**修复**：`LimitStorage_Read()` 后加范围校验，温度限超出 -40~125°C 重置为 35，光照限超出 0~100% 重置为 50。
